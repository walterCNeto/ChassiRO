§ I.i

Por que existir.

Mapear o que se aplica é trabalho que toda instituição faz, mal, em silêncio, e com gente sênior demais para isso.

A regulação financeira brasileira é uma máquina de produzir normas. CMN, BCB, CVM, SUSEP, COAF, ANPD, B3, BSM, ANBIMA — cada um com seu próprio instrumento, sua própria periodicidade, sua própria forma de revogar coisas tacitamente. O resultado é uma planilha esquecida em cada banco, em cada seguradora, em cada gestora — desatualizada na semana seguinte à publicação.

O Chassi Universal de RCSA existe para resolver isso de uma única vez. Um catálogo curado, versionado em SemVer, expondo normas, processos, riscos e seus vínculos qualificados — o tipo de ativo que uma indústria inteira precisa, mas que ninguém consegue justificar manter sozinho.

Não somos um SaaS de RCSA. Não armazenamos seu CNPJ, sua estrutura, suas posições, sua estratégia. Recebemos atributos. Devolvemos a fatia aplicável. Nada mais transita. Por design.

O moat é o catálogo, não o código. E catálogo é trabalho — não commodity.

§ IV — Em prática

Quatro linhas para a fatia aplicável.

# Banco múltiplo S2 com câmbio comercial e crédito PJ
curl https://api.chassi.com.br/v1/instancia \
  -H "X-API-Key: $CHASSI_KEY" \
  -G \
  -d "tipo_entidade=ENT_BANCO_MULTIPLO" \
  -d "segmento=S2" \
  -d "atividades=ATV_CAMBIO_COMERCIAL,ATV_CREDITO_PJ"
from chassi import Client

cli = Client(api_key=os.environ["CHASSI_KEY"])

fatia = cli.instancia(
    tipo_entidade="ENT_BANCO_MULTIPLO",
    segmento="S2",
    atividades=["ATV_CAMBIO_COMERCIAL", "ATV_CREDITO_PJ"],
)

print(f"{len(fatia.normas_aplicaveis)} normas, {len(fatia.processos_relevantes)} processos")
# → 18 normas, 73 processos
import { Chassi } from "@chassi/sdk";

const cli = new Chassi({ apiKey: process.env.CHASSI_KEY });

const fatia = await cli.instancia({
  tipo_entidade: "ENT_BANCO_MULTIPLO",
  segmento:      "S2",
  atividades:    ["ATV_CAMBIO_COMERCIAL", "ATV_CREDITO_PJ"],
});

console.log(fatia.normas_aplicaveis.length); // 18
-- consulta direta no snapshot SQLite
SELECT n.id, n.tipo, n.numero, n.ano, n.ementa
FROM   normas n
WHERE  n.status = 'vigente'
  AND (
    NOT EXISTS (SELECT 1 FROM norma_aplicabilidade_tipo_entidade
                WHERE norma_id = n.id)
    OR EXISTS  (SELECT 1 FROM norma_aplicabilidade_tipo_entidade
                WHERE norma_id = n.id AND tipo_entidade_id = 'ENT_BANCO_MULTIPLO')
  )
  AND (
    NOT EXISTS (SELECT 1 FROM norma_aplicabilidade_segmento
                WHERE norma_id = n.id)
    OR EXISTS  (SELECT 1 FROM norma_aplicabilidade_segmento
                WHERE norma_id = n.id AND segmento_id = 'S2')
  );
§ V — Anatomia

Hierarquia paralela — P × R.

Processos e riscos andam lado a lado.

O chassi mantém duas hierarquias paralelas: P0 → P4 de processos e R0 → R4 de riscos. Controles alocam no nível mais granular do risco e agregam para cima através de ambas as hierarquias simultaneamente.

Cada nó tem um nucleo — universal, bancário, mercado de capitais, seguros, previdência, capitalização ou conglomerado — que governa a quem aquele nó se aplica.

Vínculos qualificados.

Norma e processo não se relacionam de forma binária. Três níveis distinguem cobertura efetiva de cobertura nominal: primária (o processo é dono do cumprimento), secundária (contribui), informativa (princípios apenas).

Aplicabilidade declarativa.

Cada norma carrega listas explícitas de tipos_entidade, segmentos e atividades aos quais se aplica. Vazio significa sem restrição. Um banco S2 com câmbio recebe uma fatia diferente de uma DTVM, sem ambiguidade.

§ VII — Perguntas honestas

O que costumam perguntar.

Vocês são um RCSA SaaS?
Não. Somos um catálogo regulatório consumível. RCSA é o que você faz — nós entregamos a base normativa, a árvore de processos e a árvore de riscos, qualificadas e versionadas, para que sua ferramenta de RCSA (interna ou de terceiros) pare de gastar esforço no que é commodity.
Vocês armazenam meus dados?
Não armazenamos dados de cliente. A API é passiva: você passa atributos categóricos (tipo de entidade, segmento, atividades canônicas) e nós devolvemos a fatia. Não recebemos CNPJ, posição, estratégia, controles. Telemetria operacional registra apenas volume de chamadas — nunca o conteúdo das queries.
Quem mantém a curadoria?
A WCN Softwares, com revisão periódica e roadmap público. Cada release é assinado, versionado em SemVer e acompanhado de changelog estruturado. Em tier Pro/Enterprise o cliente recebe webhooks quando uma norma é alterada, revogada ou adicionada.
O snapshot tem o mesmo conteúdo da API?
Sim. Snapshot e API consomem o mesmo schema canônico — só mudam a modalidade de entrega. JSON e SQLite são gerados na mesma transação que serve a API, garantindo paridade.
O conteúdo da v0.1.0 é definitivo?
Não. A v0.1.0 cobre as normas estruturantes do SFN/SNSP — cerca de 30 normas. A v0.2 expande para top-150 normas BACEN incluindo cartas-circulares, e adiciona detalhe P3/P4 com controles-modelo. Numeração, datas e status devem ser revalidados contra a base oficial dos reguladores antes do uso operacional.
Vocês competem com Refinitiv, Thomson Reuters ou similares?
Não na mesma camada. Aqueles produtos vendem feeds de regulação global, em inglês, em formato textual. O Chassi é um modelo de dados em português, especializado no aparato regulatório brasileiro, com vínculos qualificados a processos e riscos — algo que feeds textuais não entregam.
